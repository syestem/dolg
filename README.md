# Трекер долгов

Статический одностраничный трекер долгов на HTML, CSS и ванильном JavaScript. Работает из `file://`, на GitHub Pages и без внешних CDN.

## Запуск локально

Откройте `index.html` двойным кликом в браузере. Для `file://` приложение автоматически использует локальный fallback `js/file-app.js`, потому что браузеры часто блокируют загрузку ES-модулей напрямую с файловой системы. На GitHub Pages и локальном HTTP-сервере используется ES-модульный entrypoint `js/main.js`.

Данные хранятся в `localStorage`, поэтому после перезагрузки состояние сохраняется на этом же устройстве и в этом же браузере.

## Структура

- `index.html` — единственная страница приложения.
- `styles.css` — адаптивный темный интерфейс.
- `js/state.js` — состояние, `localStorage`, CRUD и импорт.
- `js/github.js` — опциональная синхронизация через GitHub Contents API.
- `js/ui.js` — рендер, формы, модальные окна и обработчики.
- `js/audit.js` — журнал аудита и форматирование изменений.
- `js/analytics.js` — расчеты остатков, сводки и аналитики.
- `js/file-app.js` — classic fallback для запуска двойным кликом из `file://`, сгенерированный из модулей.
- `data.example.json` — пример структуры данных.
- `.nojekyll` — отключение Jekyll на GitHub Pages.

## GitHub Pages

1. Закоммитьте файлы в репозиторий.
2. Откройте репозиторий на GitHub.
3. Перейдите в `Settings` → `Pages`.
4. В `Build and deployment` выберите `Deploy from a branch`.
5. В `Branch` выберите нужную ветку, например `main`, и папку `/root`.
6. Нажмите `Save`.
7. Откройте URL, который покажет GitHub Pages.

Все пути в приложении относительные, поэтому оно работает из корня Pages-репозитория без сборки.

## Бэкап JSON

- `Экспорт JSON` скачивает текущее состояние приложения.
- `Импорт JSON` может заменить состояние или слить файл с текущими данными.
- При битом JSON приложение показывает ошибку и не падает.

## Опциональная GitHub-синхронизация

В настройках приложения можно включить синхронизацию и указать:

- `owner` — владелец репозитория;
- `repo` — имя репозитория;
- `branch` — ветка, обычно `main`;
- `path` — путь к файлу, например `data.json`;
- `Personal Access Token` — fine-grained token с доступом к Contents нужного репозитория.

Токен и параметры синхронизации хранятся только в `localStorage` браузера. Не добавляйте токен в исходники, `README`, коммиты или публичный `data.json`: публичный GitHub Pages-репозиторий не скрывает код и файлы от посетителей.

Для чтения публичного репозитория токен может быть не нужен. Для записи через GitHub Contents API токен обязателен.

## Ручная проверка

1. Откройте `index.html` локально.
2. Создайте несколько долгов с разными категориями и банками.
3. Добавьте платежи и начисления, проверьте остатки, переплату и прогресс.
4. Отредактируйте и удалите одну операцию.
5. Закройте и переоткройте долг.
6. Добавьте, переименуйте и попробуйте удалить используемый элемент справочника.
7. Проверьте сводку, аналитику и журнал аудита.
8. Экспортируйте JSON, затем импортируйте его в режиме замены или слияния.
9. Перезагрузите страницу и убедитесь, что данные сохранились.
10. Откройте консоль браузера и убедитесь, что ошибок нет.
